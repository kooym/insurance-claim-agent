# 보상 직원 워크플로우 (Phase 2)

> **대상**: 보험금 지급·심사 직원  
> **입력**: 청구 서류 폴더(`data/sample_docs/{청구번호}/` 등)  
> **출력**: `outputs/{청구번호}/` 산정 결과

---

## 1. 시스템이 수행하는 절차

| 순서 | 단계 | 내용 |
|------|------|------|
| 1 | 문서 파싱 | `.txt` 서류 → 유형별 필드 추출. **진료비세부내역서**에서 항목코드·비급여 금액 추출(`billing_items`). |
| 2 | 컨텍스트 조립 | `ClaimContext` — KCD, 입원일수, 급여/비급여, 청구 담보(IND/SIL/SUR), `billing_items` 병합. |
| 3 | 룰 엔진 | COM → DOC → IND/SIL/SUR → FRD 순. 실패 시 부지급·보류로 조기 종료. |
| 4 | 결과 문서 | `지급결의서.txt`, `고객안내문.txt`, **`보상직원_산정요약.txt`**, `decision.json`, `처리로그.json`. |

---

## 2. 직원이 꼭 읽을 파일

1. **`보상직원_산정요약.txt`** — 워크플로우, 근거 문서 경로,보별 산식·금액, 지급 예정일(3영업일), 검토 플래그.  
2. **`decision.json`** — `expected_payment_date`, `payment_deadline_note` 포함.  
3. 필요 시 **`처리로그.json`** — 룰별 evidence 전체.

---

## 3. Phase 2 산정 규칙 요약

| 영역 | 규정 요약 |
|------|-----------|
| **입원일당** | 질병: 입원일 − 4일 지급. 재해(KCD V/W/X/Y): −1일. 0일이면 IND 부지급 처리. |
| **실손 4세대** | 세부내역서 `billing_items` 있으면 비급여 3대 항목(도수·주사·MRI) **연간 한도** 반영. 없으면 총액만 사용(안내 문구). |
| **수술비** | 수술명·코드 미매칭 시 **KCD→분류표 후보**로 추정 가능 → 금액 산출하되 **검토필요**·담당자 확인. |
| **지급 예정일** | 접수일 기준 **3영업일 후** 날짜(토·일·주요 공휴일 제외). |

---

## 4. 실행 방법

```bash
source .venv/bin/activate
python -m src.agents.orchestrator   # 전체 시나리오
```

단일 건은 Python에서 `process_claim(claim_id, policy_no, claim_date)` 호출.
