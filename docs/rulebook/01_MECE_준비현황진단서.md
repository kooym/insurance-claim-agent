---

## 제1원칙: 보험금 산정의 필수 구성 요소

```
보험금 산정 = f(A 입력 서류, B 참조 데이터, C 판단 로직, D 출력 기준)

이 4개 축 중 하나라도 비어있으면 → 산정 불가 또는 결과 신뢰 불가
```

## 축 A: 입력 서류 (고객이 내는 것)

> 제1원칙: "Agent가 읽을 실제 파일이 물리적으로 존재하는가"

### 서류 완비 현황

| 시나리오 | 청구서 | 진단서 | 입원확인서 | 영수증 | 세부내역서 | 수술확인서 | 완비 |
|----------|--------|--------|------------|--------|------------|------------|------|
| CLM-001 홍길동 | ✅ | ✅ | ✅ | ✅ | ✅ | — | ✅ **완비** |
| CLM-002 김영희 | ✅ 신규 | ✅ 신규 | ❌ 의도적 미제출 | ✅ 신규 | ✅ 신규 | ✅ | ✅ **완비** |
| CLM-003 이철수 | ✅ | ✅ | ✅ 신규 | ✅ 신규 | ❌ 미생성 | — | ⚠️ |
| CLM-004 박지수 | ✅ | ✅ | — | ❌ 의도적 | ❌ 의도적 | ❌ 의도적 | ✅ **의도적 미비 (보류 시나리오)** |
| CLM-005 최민준 | ✅ 신규 | ✅ 신규 | ✅ 신규 | ✅ 신규 | ✅ 신규 | — | ✅ **완비** |

### 잔여 미완성 (지금 당장 보완 필요)
```
⚠️ CLM-003 세부내역서 — 진료비 세부내역서 텍스트 파일 미생성
   → 부지급 케이스이므로 산정에 직접 영향 없으나
     "Agent가 서류를 읽고 판단하는" 흐름 완성을 위해 필요
   → 우선순위: 중 (부지급 판정에는 진단서만 있어도 가능)
```

## 축 B: 참조 데이터 (Agent가 조회해야 하는 내부 DB)

> 제1원칙: "룰을 실행하는 데 필요한 모든 데이터 테이블이 존재하는가"

### 이번에 생성된 참조 데이터

| 파일 | 내용 | 상태 |
|------|------|------|
| `contracts_db.json` | 5개 계약 정보 (가입 담보, 가입금액, 세대, 납입 상태) | ✅ 완료 |
| `claims_history_db.json` | 5인 청구 이력 (연간 누적, 과거 이력, FRD 플래그) | ✅ 완료 |
| `kcd_exclusion_map.json` | KCD 면책코드 (알코올 K70.x, 마약 F11-F19, 자해 X60-X84 등) | ✅ 완료 |
| `surgery_classification.json` | 수술 분류표 50+ 항목 (1~5종 매핑, 비수술 시술 목록) | ✅ 완료 |
| `silson_generation_map.json` | 실손 세대별 자기부담금 산정 공식 (1~4세대) | ✅ 완료 |

### 여전히 부재한 참조 데이터 (PoC 외 영역)

| 데이터 | 필요 이유 | 우선순위 | 대안 |
|--------|-----------|----------|------|
| 비급여 평균 단가 (심평원 공시) | FRD-004 (비급여 과다 판단) | 낮음 | 60% 임계값으로 대체 가능 |
| 블랙리스트 의료기관 목록 | FRD-002 | 낮음 | 목업에서 빈 리스트 처리 |
| 장해 등급 판정 기준 (AMA) | 후유장해 산정 시 필요 | 없음 (PoC 범위 외) | — |
| 암 특약 지급 기준 | 암 진단금 산정 | 없음 (PoC 범위 외) | — |

## 축 C: 판단 로직 (룰/약관)

> 제1원칙: "모든 시나리오의 판정 경로가 룰로 커버되는가"

### 시나리오별 룰 커버리지

| 시나리오 | 결정 경로 | 필요 룰 | 룰 정의 완성도 |
|----------|-----------|---------|----------------|
| CLM-001 지급 | COM통과 → IND산정 → SIL산정 | COM-001~003, IND-001~002, SIL-000~002 | ✅ 100% |
| CLM-002 지급+플래그 | COM통과 → SUR산정 → SIL산정 → FRD-004 | COM, SUR-001, SIL-001, FRD-004 | ✅ 100% |
| CLM-003 부지급 | COM-003 FAIL → 즉시 부지급 | COM-003 (K70.3 감지) | ✅ 100% |
| CLM-004 보류 | 서류 체크리스트 FAIL → 보류 | 필수서류 체크 룰 | ✅ 100% |
| CLM-005 부지급 | COM-002 FAIL → 부지급 (63일 < 90일) | COM-002 (면책기간) | ✅ 100% |

### 실손 세대 분기 — 이번에 추가됨

```
CLM-001, CLM-002, CLM-004 → 3세대 (계약일 기준)
  공식: (급여본인부담 + 비급여) × 80%

CLM-003, CLM-005 → 4세대 (계약일 기준)
  공식: 급여본인부담 × 80% + 비급여 × 70% (단, CLM-003은 부지급이므로 미적용)
```

### 룰 간 연결 검증 (빠진 연결고리 없음)

```
청구서 → 계약번호 → contracts_db.json 조회 → 가입 담보 + 세대 확인
진단서 → KCD코드 → kcd_exclusion_map.json 조회 → 면책 여부
수술확인서 → 수술코드 → surgery_classification.json 조회 → 수술 종류
영수증 → 급여/비급여 금액 → silson_generation_map.json 공식 → 지급액
이력 조회 → claims_history_db.json → 연간 한도 + FRD 체크
```

## 잔여 보완 항목 (우선순위 정렬)

### 반드시 해야 할 것 (착수 블로커)
```
없음. 현재 상태로 코드 개발 착수 가능.
```

### 품질을 위해 하면 좋은 것
```
우선순위 HIGH:
  [ ] CLM-003 세부내역서 파일 생성
      → 3~5분 작업, 부지급 흐름 완결성을 위해

  [ ] 반자동 처리 결의서 샘플 생성 (CLM-002 담당자 검토 플래그 케이스)
      → 담당자가 승인/반려하는 UI 흐름 설계 시 필요

우선순위 MEDIUM:
  [ ] 변호사선임비용 시나리오 서류 생성 (CLM-2024-006)
      → 불송치결정서, 변호사 세금계산서 목업 필요
      → 폴더 내 기존 특약 파일 공유 받으면 즉시 진행 가능

  [ ] 비급여 평균 단가 기준 목업 테이블
      → FRD-004 룰의 "2배 초과" 판단을 위한 기준선 데이터

우선순위 LOW (PoC 2차 이후):
  [ ] 암 진단금 시나리오 (CLM-004 박지수가 암 진단이므로 확장 가능)
  [ ] 장해 등급 판정 시나리오
  [ ] 구실손(1~2세대) 시나리오 추가
```
