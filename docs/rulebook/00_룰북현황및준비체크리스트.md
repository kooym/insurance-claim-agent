## 2. 개발 착수 전 필수 준비 항목

### Phase 0 — 지금 당장 해야 할 것

```
□ [DATA-01] "자동차사고변호사선임비용특별약관" 파일 경로 확인
    → 파일 위치를 알려주시면 즉시 내용 분석 후 룰북에 반영

□ [DATA-02] 수술분류표 전체 입력
    → 현대해상 약관 별표2 수술분류표 전체를 data/policies/수술분류표.json으로 입력
    → 현재 12개 수술만 있음, 수백 개 항목 전체 필요

□ [DATA-03] KCD 면책 코드 전체 목록 완성
    → 법무팀과 함께 면책 KCD 코드 전체 확정
    → 현재: K70.x, F10.x 등 일부만 포함

□ [DATA-04] 계약 정보 목업 DB 설계
    → 각 시나리오별 계약 정보 (가입 담보, 가입금액, 계약일) 구조 설계
    → data/test_cases/contracts.json 파일로 관리 예정
```

### Phase 1 — PoC 범위 내 필요한 것

```
□ [RULE-01] 실손 세대별 분기 로직 구현
    → 계약일 입력 → 1/2/3/4세대 자동 판별 → 해당 자기부담금 적용

□ [RULE-02] 수술분류 LLM 보완 로직
    → 매핑 테이블에 없는 수술명 → LLM에게 분류 요청 + 담당자 플래그

□ [RULE-03] 부상등급표 자동 변환
    → 진단서 "전치 N주" → 1~14급 자동 변환 테이블 (변호사선임비용 룰 필요)

□ [RULE-04] 중대법규위반 자연어 판단
    → 불송치결정서/불기소고지서 텍스트에서 위반 조항 추출 (LLM)
```

### Phase 2 — PoC 완성도 향상을 위해

```
□ [DATA-05] 과거 청구 이력 목업 DB
    → FRD 룰 (반복 청구, 연간 한도 누적) 적용을 위해 필요
    → 목업으로 5~10건 가상 이력 데이터 생성

□ [DATA-06] 건강보험심사평가원 비급여 평균 데이터
    → FRD-004 (비급여 과다) 룰 적용을 위해 필요
    → 공개 API 또는 엑셀 데이터 활용

□ [RULE-05] 변호사선임비용 계약일 분기
    → 2025.10 이전: Hi2304 기준
    → 2025.10 이후: 심급별 차등 기준
```
