# PoC 시나리오 정의서 — "파일 입력 → 산정 결과 출력" 실증 흐름

> **핵심 컨셉**: 담당자가 고객 제출 서류(PDF/텍스트)를 폴더에 넣으면,
> Agent가 서류를 읽고 → 약관과 대조하고 → 보험금을 산정해서 → 결과 문서를 자동 생성한다.

---

## 실증 데모 전체 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│  [담당자 액션]  고객 청구 서류를 시스템에 업로드                        │
│   data/sample_docs/CLM-2024-001/                               │
│     ├── 01_보험금청구서.txt                                        │
│     ├── 02_진단서.txt                                             │
│     ├── 03_입원확인서.txt                                          │
│     ├── 04_진료비영수증.txt                                        │
│     └── 05_진료비세부내역서.txt                                     │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓ Agent 자동 실행
┌─────────────────────────────────────────────────────────────────┐
│  [STEP 1] 문서 처리 Agent                                         │
│   - 파일 목록 스캔, 서류 종류 자동 분류                                │
│   - 각 서류에서 핵심 정보 추출 (LLM + 정규식)                         │
│                                                                 │
│   추출 결과 예시:                                                   │
│   {피보험자: "홍길동", KCD코드: "K35.8", 입원일수: 5,                  │
│    진료비: {급여본인부담: 350000, 비급여: 180000}}                    │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  [STEP 2] 약관 검색 Agent (RAG)                                   │
│   - KCD코드, 수술명을 쿼리로 약관에서 관련 조항 검색                    │
│                                                                 │
│   검색 예시:                                                       │
│   쿼리: "K35.8 급성 충수염 입원 보장 여부"                             │
│   결과: "제1조 제1항 - 질병으로 인한 입원, 제3조 면책기간 90일"           │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  [STEP 3] 룰 엔진 Agent                                           │
│   - 공통 룰 순차 적용 (계약유효성 → 면책기간 → 면책사유)                 │
│   - 담보별 산정 룰 적용                                              │
│   - 보험금 계산                                                     │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  [STEP 4] 결과 생성 Agent                                         │
│   - 지급결의서 자동 작성 (산정 근거 + 약관 조항 인용 포함)               │
│   - 고객 안내문 자동 작성                                            │
│   - 감사 로그 저장                                                  │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│  [담당자 검토 화면]                                                  │
│   outputs/CLM-2024-001/                                         │
│     ├── 지급결의서.txt        ← Agent 작성, 담당자 최종 서명            │
│     ├── 고객안내문.txt        ← 발송용 안내문                          │
│     └── 처리로그.json         ← 전체 판단 근거 추적                    │
└─────────────────────────────────────────────────────────────────┘
```

## 시나리오별 입력 파일 명세

### 시나리오 1: 표준 지급 (입원일당 + 실손) — CLM-2024-001

**청구 인물**: 홍길동 (45세, 남 / 계약 4년차 / 급성 맹장염)
**핵심 검증 포인트**: Agent가 5개 서류를 읽고 정확히 574,000원을 산정하는가

| 파일명 | 서류 종류 | Agent가 추출해야 할 핵심 정보 |
|--------|-----------|-------------------------------|
| `01_보험금청구서.txt` | 보험금청구서 | 청구번호, 계약번호, 청구 담보 종류, 서명일 |
| `02_진단서.txt` | 진단서 | 진단명, KCD코드(K35.8), 진단일, 의사명, 의료기관 |
| `03_입원확인서.txt` | 입원확인서 | 입원일(11/10), 퇴원일(11/14), 입원일수(5일) |
| `04_진료비영수증.txt` | 진료비영수증 | 총액(1,250,000원), 급여본인부담(350,000원), 비급여(180,000원) |
| `05_진료비세부내역서.txt` | 세부내역서 | 항목별 비용, 급여/비급여 구분, 비급여 항목 내용 |

**예상 Agent 판단 흐름**:
```
문서 분류 완료 → 5개 서류 모두 인식
계약 조회: POL-20200315-001 → 유효, 입원일당(30,000원/일) + 실손 가입 확인
면책기간 체크: 계약일(2020.03.15) → 사고일(2024.11.10) = 약 4년 7개월 → 통과
면책사유 체크: K35.8 (급성충수염) → 면책 목록 없음 → 통과
입원일당 산정: 5일 × 30,000원 = 150,000원
실손 산정: (350,000 + 180,000) × 80% = 424,000원
최종 지급액: 574,000원
담당자 플래그: 없음 (자동 처리)
```

### 시나리오 2: 수술비 + 비급여 과다 플래그 — CLM-2024-002

**청구 인물**: 김영희 (33세, 여 / 계약 6년차 / 유방 종양 수술)
**핵심 검증 포인트**: 비급여 비중 72% → 담당자 검토 플래그 자동 발생, 서류 미비(입원확인서 없음) 감지

| 파일명 | 서류 종류 | 특이사항 |
|--------|-----------|----------|
| `01_보험금청구서.txt` | 보험금청구서 | 수술비 + 실손 청구, 입원일당 미청구 |
| `02_진단서.txt` | 진단서 | KCD: D24, 유방 양성 종양 |
| `03_수술확인서.txt` | 수술확인서 | 수술명: 유방 종양 절제술, 수술코드: 8510, 3종 |
| `04_진료비영수증.txt` | 진료비영수증 | 총 3,800,000원 / 비급여 1,100,000원 (72%) |
| `05_진료비세부내역서.txt` | 세부내역서 | 비급여 항목: 고가 봉합재, 수술재료대 포함 |

**예상 Agent 판단 흐름**:
```
문서 분류: 입원확인서 없음 감지 → "입원일당 청구 없음, 실손/수술비만 처리"로 자동 전환
수술 종류 판단: 수술코드 8510 → 별표2 조회 → 3종 확인 → 500,000원
비급여 비중 계산: 1,100,000 / 1,520,000 = 72.4% → 임계값(60%) 초과
→ [담당자 검토 플래그 ON] "비급여 항목 상세 확인 필요"
실손 산정: (420,000 + 1,100,000) × 80% = 1,216,000원
총 지급액: 500,000 + 1,216,000 = 1,716,000원 (담당자 승인 후 지급)
```

### 시나리오 3: 면책 사유 자동 부지급 — CLM-2024-003

**청구 인물**: 이철수 (58세, 남 / 계약 2년차 / 알코올성 간경변)
**핵심 검증 포인트**: KCD K70.3 감지 즉시 면책 판단, 부지급 안내문 자동 생성

| 파일명 | 서류 종류 | 특이사항 |
|--------|-----------|----------|
| `01_보험금청구서.txt` | 보험금청구서 | 입원일당 + 실손 청구 |
| `02_진단서.txt` | 진단서 | KCD: K70.3 (알코올성 간경변증) ← 핵심 |
| `03_입원확인서.txt` | 입원확인서 | 8일 입원 |
| `04_진료비영수증.txt` | 진료비영수증 | 총 2,100,000원 |
| `05_진료비세부내역서.txt` | 세부내역서 | 간질환 치료 내역 |

**예상 Agent 판단 흐름**:
```

진단서 분석: KCD K70.3 추출

→ [즉시 부지급 판정]
약관 RAG 검색: "K70 알코올 면책" → 제2조 제1항 제3호 조항 인용
결과 생성:

  - 부지급 사유: "약관 제2조 제1항 제3호 해당 (알코올성 질환)"
  - 고객 안내문: 면책 조항 전문 인용 + 이의신청 절차 안내

## Agent 출력물 명세 (각 케이스별 생성 파일)

### 지급 케이스 출력물 (시나리오 1, 2)
```
outputs/CLM-2024-001/
├── 지급결의서.txt
│     내용: 청구번호, 피보험자, 산정 내역, 적용 약관 조항,

├── 고객안내문.txt
│     내용: "보험금 지급 결정 안내" 형식의 공문,
│           지급 금액, 지급 예정일, 지급 계좌 확인 요청
│
└── 처리로그.json
      내용: 전체 Agent 처리 과정, 각 룰 적용 결과,
            RAG 검색 쿼리 및 결과, 타임스탬프

## 실증 시연 시나리오 (데모 순서 권장)

```
데모 1: 시나리오3 (부지급) 먼저 — 가장 극적이고 명확한 AI 판단 시연
         "KCD 코드 하나로 즉시 면책 판정 + 부지급 안내문 자동 생성"

데모 2: 시나리오1 (완전 자동 지급) — 정상 케이스 자동화 속도 시연
         "5개 파일 투입 → 30초 내 574,000원 산정 + 결의서 생성"

데모 3: 시나리오2 (검토 플래그) — Human-in-the-loop 시연
         "AI가 비급여 72% 감지 → 자동으로 담당자 검토 요청"

데모 4: 시나리오4 (서류 미비) — 서류 관리 자동화 시연
         "누락 서류 자동 감지 + 추가 제출 요청 공문 자동 생성"
```
