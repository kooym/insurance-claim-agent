# 실증(PoC) 데모 가이드

> 담당자가 실제로 시스템을 실증할 때 사용하는 단계별 가이드

---

## 데모 전 준비 확인

```
□ .env 파일에 OpenAI API 키 설정 완료
□ pip install -r requirements.txt 완료
□ data/sample_docs/ 아래 서류 파일 존재 확인
□ data/policies/standard_policy.md 존재 확인
```

---

## 데모 시나리오별 실행 방법

### 데모 1 — "면책 사유 자동 부지급" (가장 임팩트 있는 데모)

**담당자 액션:**
```
data/sample_docs/CLM-2024-003/ 폴더 안에
  01_보험금청구서.txt
  02_진단서.txt          ← KCD K70.3 포함
  03_입원확인서.txt
  04_진료비영수증.txt
  05_진료비세부내역서.txt
5개 파일이 들어 있습니다.
```

**시스템 실행:**
```bash
python src/agents/orchestrator.py --claim CLM-2024-003
```

**Agent 처리 과정 (터미널 출력):**
```
[문서처리 Agent] 파일 스캔 중...
  ✓ 01_보험금청구서.txt → 청구서 (계약번호: POL-20220901-003)
  ✓ 02_진단서.txt       → 진단서 (KCD: K70.3, 알코올성 간경변증)
  ✓ 03_입원확인서.txt   → 입원확인서 (8일)
  ✓ 04_진료비영수증.txt → 영수증 (총 2,100,000원)
  ✓ 05_세부내역서.txt   → 세부내역서

[약관 Agent] 약관 검색 중...
  쿼리: "K70.3 알코올성 간경변 면책"
  결과: 제2조 제1항 제3호 — 알코올 관련 질환 면책 조항 발견

⚠️  [룰 엔진] RULE-COM-003 면책사유 체크
  K70.3 ∈ 면책 KCD 코드 목록 (K70.x)
  → 즉시 [부지급] 판정

[결과 생성] 부지급 결의서 + 고객 안내문 생성 완료
  outputs/CLM-2024-003/부지급결의서및안내문.txt

처리 시간: 22초
```

**확인할 출력 파일:**
```
outputs/CLM-2024-003/
  ├── 부지급결의서및안내문.txt
  └── 처리로그.json
```

### 데모 2 — "5개 파일 → 574,000원 자동 산정"

**담당자 액션:**
```
data/sample_docs/CLM-2024-001/ 폴더 안에
5개 서류 파일을 넣고 실행
```

**시스템 실행:**
```bash
python src/agents/orchestrator.py --claim CLM-2024-001
```

**Agent 처리 과정:**
```
[문서처리 Agent]
  ✓ 진단서 → K35.8 (급성 충수염) 추출
  ✓ 입원확인서 → 입원 5일 (11/10 ~ 11/14) 추출
  ✓ 영수증 → 급여본인부담 350,000 / 비급여 180,000 추출

[약관 Agent]
  K35.8 → 일반 질병 → 면책 미해당 확인

[룰 엔진]
  계약유효성 ✓ → 면책기간(4년7개월) ✓ → 면책사유 없음 ✓
  입원일당: 5일 × 30,000 = 150,000원
  실손: (350,000 + 180,000) × 80% = 424,000원
  합계: 574,000원

[결과 생성] 지급결의서 생성 완료
처리 시간: 28초
```

**포인트**: 담당자가 별도로 계산기를 두드리지 않아도 근거와 함께 자동 산정

---

### 데모 3 — "비급여 과다 → 담당자 플래그"

**CLM-2024-002 실행 시:**
```
[룰 엔진] 비급여 비중 계산: 1,100,000 / 1,520,000 = 72.4%
⚠️  임계값(60%) 초과 → [담당자 검토 플래그 ON]
→ 자동 처리 불가, 담당자 검토 후 승인 필요

outputs/CLM-2024-002/지급결의서_검토필요.txt 생성
담당자 검토 대기 상태로 보류
```

**포인트**: AI가 모든 것을 처리하는 것이 아니라, 위험 케이스는 사람에게 넘기는 Human-in-the-loop 구조 시연

### 데모 4 — "서류 미비 자동 감지"

**CLM-2024-004 실행 시:**
```
[문서처리 Agent] 파일 스캔:
  ✓ 01_보험금청구서.txt (수술비 청구)
  ✓ 02_진단서.txt
  ✗ 수술확인서 — 없음!
  ✗ 진료비영수증 — 없음!

[룰 엔진] 수술비 청구 필수 서류 체크:
  청구서 ✓ / 진단서 ✓ / 수술확인서 ✗ / 영수증 ✗
  → [보류] 추가 서류 요청

[결과 생성] 서류보완요청 안내문 자동 생성
  ※ 사전 확인: 면책기간 정상 초과, 추후 서류 제출 시 처리 가능 안내 포함
```

## 데이터 흐름 전체 구조도

```
[입력]                [처리]                      [출력]
─────────────────────────────────────────────────────────────

data/sample_docs/     ┌─────────────────────┐    outputs/
CLM-2024-001/        │  Orchestrator         │    CLM-2024-001/
  01_보험금청구서.txt  │  (LangGraph)          │      지급결의서.txt
  02_진단서.txt       │                      │      고객안내문.txt
  03_입원확인서.txt   │  ┌──────────────┐    │      처리로그.json
  04_진료비영수증.txt │  │ 문서처리 Agent│    │
  05_세부내역서.txt   │  └──────┬───────┘    │
                     │         ↓             │
data/policies/       │  ┌──────────────┐    │
  standard_policy.md │  │ 약관검색 Agent│    │
  (→ 벡터 DB)        │  │   (RAG)      │    │
                     │  └──────┬───────┘    │
                     │         ↓             │
                     │  ┌──────────────┐    │
                     │  │ 룰엔진 Agent  │    │
                     │  └──────┬───────┘    │
                     │         ↓             │
                     │  ┌──────────────┐    │
                     │  │ 결과생성 Agent│    │
                     │  └──────┬───────┘    │
                     └─────────┼────────────┘
                               ↓
                     담당자 검토 화면
                     (Streamlit UI 예정)
```

## 각 단계별 "Agent가 하는 일" vs "사람이 하는 일"

| 단계 | Agent | 사람(담당자) |
|------|-------|-------------|
| 서류 접수 | 자동 파일 감지 및 분류 | 업로드 액션만 |
| 정보 추출 | KCD코드, 날짜, 금액 자동 추출 | 없음 |
| 약관 대조 | RAG로 관련 조항 검색 + 인용 | 없음 |
| 룰 적용 | 20개 이상 룰 자동 순차 적용 | 없음 |
| 보험금 계산 | 자동 계산 + 근거 명시 | 없음 |
| 결의서 작성 | 초안 자동 생성 | **최종 검토 + 서명** |
| 고객 안내문 | 자동 생성 (지급/부지급/보완 요청) | 발송 승인 |
| 이상 케이스 | 플래그 발생 + 근거 제시 | **직접 판단 + 처리** |

## PoC 성공 기준

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 정보 추출 정확도 | 95% 이상 | 5개 서류 × 핵심 필드 수작업 비교 |
| 산정 금액 정확도 | 100% (시나리오 기준) | 예상값과 대조 |
| 면책 판단 정확도 | 100% | 시나리오 3, 5 정확 부지급 여부 |
| 서류 미비 감지율 | 100% | 시나리오 4 감지 여부 |
| 처리 시간 (건당) | 60초 이내 | 타임스탬프 측정 |
| 담당자 플래그 정확도 | 90% 이상 | 시나리오 2 플래그 발생 여부 |
