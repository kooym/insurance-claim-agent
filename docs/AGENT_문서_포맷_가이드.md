# Agent 문서 포맷 가이드

> **목적**: AI Agent가 문서를 읽고 올바르게 해석·연결할 수 있도록 문서 작성 시 준수할 포맷 규칙.

---

## 1. 문서 상단 블록

다음 정보를 문서 최상단 인용 블록(blockquote)으로 넣는다.

```markdown
# 문서명 (Agent 구현용)

> **출처 및 법적 근거** (해당 시)
> - 출처 1
> - 출처 2
>
> **목적** (선택): 이 문서를 Agent가 언제 참조하는지 한 줄.
> **코드 연결점** (선택): 예) rule_engine.py::rule_sil()
```

- 보험 기준 문서: 출처·법적 근거 필수.
- 아키텍처·플로우: 목적·코드 연결점 권장.

---

## 2. 제목 계층

- `##` : 대분류 (1. 2. 3.)
- `###` : 중분류 (1.1, 2.1)
- `####` : 소분류 (필요 시)

제목만으로도 문서 구조가 드러나도록 한다. Agent는 제목 목차로 섹션을 찾을 수 있다.

---

## 3. 표 사용

- **필드·스키마**: `| 필드 | 타입 | 설명 |` 형태로 통일.
- **룰·연결점**: `| 룰 ID | 내용 | 코드 연결점 |` 등으로 정리.
- **상태·갭**: `| 항목 | 상태 | 갭 |` 형태 권장.

표는 파싱·요약에 유리하므로 리스트보다 우선한다.

---

## 4. 코드·데이터 연결점 명시

각 기준·룰 문서 끝에 다음 중 하나를 포함한다.

- **Agent 구현 시 참조**
- **코드 연결점**
- **연결점**: `모듈::함수` 또는 `data/reference/파일명`

예:

```markdown
## 7. Agent 구현 시 참조

- rule_engine.py::rule_sil() — 세대별 copay 적용 시 본 문서 계산식 참조.
- data: silson_generation_map.json
```

---

## 5. 입력·출력 명시

모듈·함수·플로우를 설명할 때는 다음을 명시한다.

- **입력**: 경로, 타입, 예시 값
- **출력**: 타입, 예시 또는 필드 목록

예:

```markdown
### 입력
- 경로: data/sample_docs/{claim_id}/
- 타입: 디렉터리 내 .txt 파일 목록

### 출력
- 타입: list[ParsedDocument]
- 필드: doc_type, confidence, fields
```

---

## 7. 상태·Phase 표기

- **✅ / ❌ / ⏳**: 상태 표시 시 일관 사용.
- **Phase 2**: 아직 구현되지 않은 항목은 "Phase 2" 또는 "미구현"으로 명시.

이렇게 하면 Agent가 "현재 구현된 범위"와 "추가 작업 대상"을 구분할 수 있다.
