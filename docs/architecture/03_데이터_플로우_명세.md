# 데이터 플로우 명세

> **목적**: Agent가 스키마·필드 타입을 참조해 파싱·룰 입력·출력을 맞추기 위함.  
> **범위**: ParsedDocument → ClaimContext → RuleResult → ClaimDecision 필드별 인터페이스.  
> **코드 연결점**: `src/schemas.py`, `src/ocr/doc_parser.py`, `src/agents/orchestrator.py::build_claim_context()`, `src/rules/rule_engine.py`

---

## 1. ParsedDocument

| 필드 | 타입 | 설명 |
|------|------|------|
| doc_type | str | "보험금청구서", "진단서", "입원확인서", "수술확인서" 등 |
| file_path | str | 원본 파일 경로 |
| confidence | float | 0.0~1.0 파싱 신뢰도 |
| fields | dict | 추출된 필드 (키: 필드명, 값: 추출값) |

### fields 예시

```json
{
  "accident_date": "2024-11-01",
  "admission_date": "2024-11-01",
  "discharge_date": "2024-11-05",
  "hospital_days": 5,
  "kcd_code": "K35.8",
  "diagnosis": "급성 충수염",
  "covered_self_pay": 120000,
  "non_covered": 0,
  "claimed_coverage_types": ["IND", "SIL"],
  "surgery_name": "복강경 충수절제술",
  "surgery_code": "4701"
}
```

---

## 2. ClaimContext

| 필드 | 타입 | 설명 |
|------|------|------|
| claim_id | str | 청구번호 |
| policy_no | str | 계약번호 |
| claim_date | str | 청구일 |
| accident_date | str | 사고일 |
| admission_date | str | 입원일 |
| discharge_date | str | 퇴원일 |
| hospital_days | int | 입원일수 |
| kcd_code | str | KCD 진단코드 |
| diagnosis | str | 진단명 |
| surgery_name | str | 수술명 |
| surgery_code | str | 수술코드 |
| covered_self_pay | float | 급여 본인부담금 |
| non_covered_amount | float | 비급여 금액 |
| submitted_doc_types | list[str] | 제출 서류 유형 목록 |
| claimed_coverage_types | list[str] | 청구 담보 (IND, SIL, SUR) |
| raw_documents | list[ParsedDocument] | 원본 파싱 결과 |
| parse_confidence_min | float | 최소 파싱 신뢰도 |

---

## 3. RuleResult

| 필드 | 타입 | 설명 |
|------|------|------|
| rule_id | str | COM-001, IND-001 등 |
| status | str | PASS, FAIL, SKIP, FLAGGED |
| reason | str | 판정 사유 |
| value | float | 지급액 (PASS 시) |
| evidence | dict | 판정 근거 |

---

## 4. ClaimDecision

| 필드 | 타입 | 설명 |
|------|------|------|
| decision | str | "지급", "부지급", "보류", "검토필요" |
| total_payment | float | 총 지급액 |
| breakdown | dict | 룰별 상세 (IND-001, SIL-001 등) |
| denial_reason | str | 부지급 사유 |
| policy_clause | str | 적용 약관 조항 |
| missing_docs | list[str] | 미제출 서류 |
| reviewer_flag | bool | 담당자 검토 필요 여부 |
| reviewer_reason | str | 검토 사유 |
| fraud_investigation_flag | bool | 사기조사 플래그 |
| fraud_investigation_reason | str | 사기조사 사유 |
| applied_rules | list[RuleResult] | 적용된 룰 목록 |

---

## 5. 데이터 흐름

```
parse_claim_documents()  →  list[ParsedDocument]
build_claim_context()    →  ClaimContext
run_rules()              →  ClaimDecision (내부 RuleResult 수집)
write_results()          →  파일 출력 (decision.json, 지급결의서.txt 등)
```
